# 🚀 AI Code Studio (browserAI) — Full-Stack AI-Powered IDE

**AI Code Studio** — это полноценная открытая среда разработки (IDE) со встроенным ИИ-ассистентом, агентом для автономной генерации проектов, управлением браузером через Playwright и изолированным выполнением кода в Docker (аналог Cursor, Replit Agent и Browser AI).

---

## ✨ Возможности

### 🤖 AI Интеграция
- **AI Чат с контекстом проекта:** AI видит ваши открытые файлы и структуру репозитория. Поддержка слеш-команд: `/refactor`, `/explain`, `/fix`, `/test`, `/doc`.
- **Автозаполнение кода (Inline Completion):** Умное заполнение строк (Fill-in-the-Middle) прямо в процессе набора кода.
- **AI Агент для создания проектов:** Генерация полноценного работоспособного приложения с нуля по вашему текстовому описанию (с установкой npm-пакетов и проверкой сборки).
- **Мультипровайдерность:** Поддержка OpenAI, Anthropic, Google Gemini, DeepSeek, Groq, Mistral, Qwen, а также локальных LLM через Ollama или LM Studio.

### 📝 Редактор кода (Monaco Editor)
- Полноценный редактор на базе Monaco (как в VS Code).
- Подсветка синтаксиса (TS, JS, Python, Go, Rust, HTML, CSS и др.).
- Многофайловые вкладки, автосохранение, отображение ошибок компиляции.

### 💻 Изолированный Code Runner (Docker-песочница)
- Безопасное выполнение вашего кода прямо из редактора в изолированных Docker-контейнерах.
- Поддерживаемые языки: **Python, JavaScript, TypeScript, Go, Rust**.
- Ограничение ресурсов (CPU 0.5, RAM 256MB, отключенная сеть для безопасности) и автоматическое удаление контейнера по завершении.

### 🌐 Browser AI (Playwright + MCP)
- Управление встроенным браузером прямо из ИИ-чата.
- Навигация по URL, клики по элементам, заполнение полей ввода, извлечение (парсинг) данных.
- Поддержка протокола **Model Context Protocol (MCP)** для бесшовного управления Chromium со стороны LLM.

### 👥 Коллаборация (Совместное редактирование)
- Совместная работа в реальном времени с использованием алгоритмов конфликт-разрешения **CRDT (Yjs)**.
- Отображение живых курсоров коллег и их выделения в коде.

### 🐙 Git Интеграция
- Клонирование любых Git-репозиториев (публичных и приватных через GitHub OAuth).
- Просмотр изменений (diff), создание коммитов, переключение веток, история (git log) и отправка изменений (push).

## 🚀 Быстрый старт через Docker Compose

Весь стек сервисов спроектирован так, чтобы запускаться одной простой командой.

### 1. Подготовка окружения
Клонируйте репозиторий и создайте файлы конфигурации:
```bash
# Скопируйте шаблон окружения
cp .env.example .env
```
*Отредактируйте `.env` и добавьте ваши API-ключи (например, `OPENAI_API_KEY` или `DEEPSEEK_API_KEY` для работы реального ИИ).*

### 2. Запуск всего стека
Запустите Docker Compose:
```bash
docker-compose up -d --build
```
Эта команда соберет и запустит следующие сервисы:
- `postgres` (Порт `5432`): Хранилище пользователей, проектов, метаданных.
- `redis` (Порт `6379`): Кэш, Pub/Sub и очередь задач.
- `backend` (Fastify, Порт `3000`): Основной API & WebSocket сервер.
- `code-runner` (FastAPI, Порт `8000`): Изолированный запуск кода в Docker.
- `browser-agent` (Playwright, Порт `8080`): Автоматизация браузера на сервере.
- `collaboration` (Yjs, Порт `3001`): Сервер синхронизации совместной работы.
- `frontend` (Nginx, Порт `80`): Сборка веб-интерфейса и раздача статики.

### 3. Открытие в браузере
Перейдите по адресу:
```
http://localhost
```

## 💻 Локальная разработка (без Docker)

Если вы хотите вносить изменения в код локально и запускать отдельные сервисы напрямую:

### Шаг 1. Переключение бэкенда на SQLite (опционально)
Для простоты локальной разработки без Postgres, измените `DATABASE_URL` в `.env`:
```env
DATABASE_URL="file:./prisma/dev.db"
```

### Шаг 2. Запуск Бэкенда (Fastify)
```bash
cd public/backend
npm install
npx prisma generate
npx prisma db push      # создаст sqlite базу dev.db
npm run dev             # сервер запустится на http://localhost:3000
```

### Шаг 3. Запуск Фронтенда (React + Vite)
```bash
# Из корневой папки проекта
npm install
npm run dev             # запустит клиент на http://localhost:5173 с прокси на бэкенд
```

## 🔒 Безопасность песочницы кода
Все команды пользователя, запускаемые через терминал (кнопка Run), транслируются в микросервис **Code Runner**, который:
1. Создает временную директорию с файлом кода.
2. Поднимает изолированный Docker-контейнер на базе минимального runtime соответствующего языка (например, `python:3.11-slim`).
3. Запускает код с жестким лимитом памяти (256MB) и процессорного времени (timeout 10с).
4. Запрещает сетевой доступ внутри контейнера (`network_disabled=True`).
5. Удаляет контейнер сразу после возврата вывода в Fastify бэкенд.

## ⚙️ Настройка AI провайдеров
Откройте **Settings (Шестеренка)** в верхнем правом углу:
- Для **OpenAI/Claude/DeepSeek**: Введите соответствующий API-ключ в поле и выберите модель.
- Для **Локального запуска**: Выберите `Ollama` или `LM Studio`. Убедитесь, что ваше локальное приложение запущено (по умолчанию на `http://localhost:11434` или `http://localhost:1234`), и введите название модели, загруженной на вашем компьютере (например, `llama3.2` или `deepseek-coder`).
