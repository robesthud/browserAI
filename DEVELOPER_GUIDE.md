# BrowserAI Developer Guide

Подробная документация по функциональным модулям проекта: что делает каждый блок, как он используется, где находится код и какие сценарии уже поддерживаются.

Этот файл нужен для новых разработчиков, чтобы быстро понять состав продукта и его текущее устройство.

## Руководство по функциональным модулям

Этот раздел — краткая внутренняя инструкция для разработчика. Он описывает не только «что есть», но и **зачем это нужно**, **как это работает**, **какие пользовательские сценарии поддерживаются** и **где искать реализацию в коде**.

### 1. Архитектура проекта

| Слой | Назначение | Основные файлы |
|---|---|---|
| Frontend | UI чата, auth, settings, workspace, cloud UX | `src/App.jsx`, `src/components/*`, `src/lib/*` |
| Backend | Auth, SQLite, vault, workspace API, web tools, раздача `dist/` | `server/index.js`, `server/db.js`, `server/workspace.js`, `server/web.js` |

### 2. Пользовательский контур: аккаунт, сессия, восстановление доступа

#### Назначение
Модуль отвечает за доступ к приложению, защиту пользовательских данных, привязку чатов и настроек к конкретному аккаунту и восстановление доступа при утере пароля.

#### Что умеет

| Функция | Что делает |
|---|---|
| Регистрация | Создаёт аккаунт по `email + password`, опционально принимает `name` и `phone` |
| Первый пользователь = owner | Самый первый зарегистрированный пользователь получает роль `owner` |
| Закрытая регистрация после первого пользователя | Для следующих регистраций требуется `REGISTRATION_SECRET` |
| Логин | Создаёт серверную сессию и устанавливает HttpOnly cookie |
| Logout | Удаляет активную сессию |
| `/api/auth/me` | Возвращает текущего пользователя по cookie |
| Обновление телефона | Даёт залогиненному пользователю сменить привязанный номер |
| Сброс пароля по email | Отправляет reset-ссылку на `APP_URL/?reset_token=...` |
| Сброс пароля по SMS | Отправляет 6-значный код, затем выдаёт временный `resetToken` |
| Сброс всех старых сессий после reset password | После смены пароля все предыдущие входы становятся невалидными |

#### Требования к паролю

Пароль должен содержать:
- минимум 10 символов;
- 1 заглавную букву;
- 1 строчную букву;
- 1 цифру;
- 1 спецсимвол.

#### Пользовательские режимы auth-экрана
Frontend поддерживает режимы:
- вход;
- регистрация;
- сброс по email;
- сброс по SMS;
- ввод нового пароля по reset token.

Дополнительно в auth-формах есть:
- кнопки показа/скрытия пароля;
- подтверждение нового пароля;
- подсказки по требованиям к паролю.

#### Где смотреть реализацию
- backend: `server/index.js`
- frontend: `src/components/AuthGate.jsx`
- клиент API: `src/lib/backend.js`

### 3. Сессии и безопасность входа

#### Как это устроено
- cookie: `browserai_session`;
- cookie: `HttpOnly`, `SameSite=Lax`, в production — `Secure`;
- срок жизни: 30 дней;
- в БД хранится **hash токена**, а не сам токен;
- для auth-эндпоинтов включён отдельный rate limit;
- чувствительные маршруты (`settings`, `keys`, `vault`, `workspace`, `cloud`) требуют входа.

#### Какие таблицы участвуют
- `users`
- `sessions`
- `password_reset_tokens`
- `sms_codes`
- `user_cloud_data`

#### Для чего это нужно
Это делает BrowserAI не просто локальным чатом, а аккаунтным приложением, где один и тот же пользователь может работать с одними и теми же чатами и настройками на нескольких устройствах.

### 4. Чаты и история диалогов

#### Назначение
Модуль реализует постоянную историю разговоров и рабочий UX чата, похожий на привычные AI-интерфейсы.

#### Что умеет

| Функция | Описание |
|---|---|
| Новый чат | Создаётся через sidebar |
| Выбор чата | Любой чат можно открыть как активный |
| Удаление чата | Удаляется из списка и из `localStorage` |
| Автозаголовок | Название строится по первому сообщению пользователя |
| Стриминг ответа | Ответ модели может приходить по частям |
| Остановка ответа | Во время стрима можно остановить генерацию |
| Ошибки ответа | Ошибки модели показываются внутри сообщения ассистента |
| Копирование ответа | У сообщения ассистента есть кнопка copy |
| Вложения в сообщении | Файлы отображаются как attachments |

#### Хранение
- чаты сохраняются в `localStorage`;
- структура: `id`, `title`, `createdAt`, `updatedAt`, `summary`, `summarizedUntil`, `messages[]`.

#### Где смотреть реализацию

- список сообщений: `src/components/MessageList.jsx`

### 5. Контекст диалога и автосуммаризация

#### Назначение
Этот блок уменьшает стоимость длинных диалогов и не даёт контексту бесконтрольно разрастаться.

#### Что делает
- формирует окно последних релевантных сообщений;
- ограничивает число сообщений и общий объём контекста;
- при длинном чате вызывает отдельную AI-сводку прошлой части диалога;
- сохраняет summary в данных чата;
- при следующем сообщении подмешивает summary в system prompt.

#### Результат для пользователя
- длинные диалоги продолжаются устойчивее;
- модель не обязана каждый раз читать всю историю целиком;
- уменьшается расход токенов.

#### Где смотреть реализацию
- `src/lib/useChats.js`
- `src/lib/api.js` (`summarizeConversation`)

### 6. AI-движок и работа с моделью

#### Назначение
Это ядро продукта: BrowserAI работает как универсальный клиент к OpenAI-compatible моделям.

#### Какие операции выполняет AI

| Функция | Что делает |
|---|---|
| Обычный чат | Отвечает на сообщения пользователя |
| Streaming | Отдаёт токены постепенно |
| Non-streaming | Возвращает полный ответ одним блоком |
| System prompt | Учитывает глобальный системный промпт |
| Temperature | Управляет вариативностью ответов |
| Memory summary | Учитывает summary старых сообщений |
| Web context | Может использовать web-контекст при включённом `useWebAI` |
| AI Create File | Генерирует содержимое нового файла |
| AI Patch File | Возвращает patch-инструкции для точечного редактирования файла |
| Validate key | Проверяет API-ключ и подбирает рабочую модель |

#### Важно для разработчика
Обычные chat-запросы идут **напрямую с frontend в указанный `baseUrl` провайдера**. Express-сервер не является обязательным прокси для chat completion.

#### Что AI не делает в текущем проекте
- не выполняет shell-команды;
- не работает как полноценный автономный агент;
- не редактирует бинарные файлы;
- не имеет встроенного image-generation workflow;
- не имеет отдельной planner/executor системы.

#### Где смотреть реализацию
- `src/lib/api.js`
- `src/lib/settings.js`
- `src/lib/useSettings.js`

### 9. Web AI: поиск и web-контекст

#### Назначение
Позволяет модели отвечать с опорой на свежую информацию из интернета, а не только на историю чата и системный промпт.

#### Какие функции выполняет модуль

| Функция | Описание |
|---|---|
| Web toggle | Включение / выключение режима в topbar |
| Search query build | Берёт последнее сообщение пользователя как поисковый запрос |
| Search | Выполняет поиск через DuckDuckGo HTML |
| Page fetch | Загружает содержимое найденных страниц через backend |
| Context assembly | Собирает краткий web-context по верхним результатам |
| Prompt enrichment | Добавляет web-context в system prompt перед chat completion |

#### Что пользователь получает
- возможность спросить о более актуальной информации;
- компактный контекст из нескольких web-страниц;
- без необходимости вручную искать ссылки и вставлять тексты.

#### Ограничения
- это не полноценный браузерный агент;
- нет интерактивного browsing flow по страницам;
- нет встроенной разметки обязательных citations в каждом ответе.

#### Где смотреть реализацию

### 10. Cloud sync между устройствами

#### Назначение
Даёт пользователю единое состояние между desktop-браузером и Android WebView.

#### Что синхронизируется
- чаты;
- `systemPrompt`;
- `temperature`;
- `stream`;
- `useWebAI`;
- `activeKeyId`;
- резервная копия ключей внутри payload.

#### Как это работает
1. После логина frontend вызывает `/api/cloud`.
2. Ответ записывается в локальное состояние / `localStorage`.
3. После изменений настройки и чаты с debounce отправляются обратно.
4. Во время streaming sync временно не уходит, чтобы не сохранять промежуточный ответ.

#### Безопасность
- данные лежат в `user_cloud_data`;
- payload шифруется на сервере через `AES-256-GCM`;
- ключ derive-ится из `AUTH_SECRET`.

#### Где смотреть реализацию
- `src/App.jsx` (`CloudSync`)
- `src/components/AuthGate.jsx`
- `src/lib/backend.js`
- `server/index.js`

### 11. Vault: шифрование API-ключей

#### Назначение
Позволяет хранить ключи провайдеров в SQLite не в открытом виде, а под мастер-паролем пользователя.

#### Поддерживаемые функции

| Функция | Описание |
|---|---|
| Включить шифрование | Перешифровывает существующие ключи |
| Разблокировать vault | Ввод мастер-пароля и загрузка ключа в память процесса |
| Заблокировать vault | Сбрасывает мастер-ключ из памяти |
| Сменить мастер-пароль | Перешифровывает все ключи новым derive key |
| Отключить шифрование | Возвращает хранение ключей в открытом виде |
| Автоблокировка | Сбрасывает доступ к ключам после бездействия |
| Backup | Экспортирует ключи и параметры |
| Restore | Восстанавливает ключи и параметры из backup JSON |

#### Технически
- шифрование: `AES-256-GCM`;
- derive function: `scrypt`;
- в БД лежат `salt`, `verifier`, `vault_enabled`;
- мастер-ключ хранится только в RAM процесса.

#### Где смотреть реализацию
- UI: `src/components/SettingsModal.jsx`
- client state: `src/lib/useSettings.js`
- backend: `server/index.js`, `server/db.js`, `server/crypto.js`

### 12. Workspace: файловая рабочая область

#### Назначение
Workspace — это встроенный файловый менеджер и мини-рабочее окружение проекта. Он нужен для хранения материалов, генерации файлов через AI и точечного редактирования кода/текста без выхода из приложения.

#### Основные пользовательские функции

| Категория | Что умеет |
|---|---|
| Навигация | открыть / скрыть workspace, дерево файлов и папок |
| Поиск | поиск по имени, grep-поиск по содержимому |
| CRUD | создать папку, создать файл, переименовать, переместить, удалить |
| Upload | загрузка файлов, загрузка папки, загрузка по URL |
| Archives | авто-распаковка `zip`, `tar`, `tgz`, `tar.gz` |
| Preview | просмотр text / code / markdown / image / pdf |
| Editing | встроенное редактирование и сохранение текстовых файлов |
| History | просмотр списка ревизий и восстановление любой ревизии |
| AI | AI Create File, AI Apply Patch |
| Chat bridge | прикрепление файла из workspace в composer |
| Download | скачивание файла или папки (папка — ZIP) |
| DnD | drag-and-drop перемещение по дереву |

#### Что показывает preview
- **Text** — plain text
- **Code** — с лёгкой подсветкой синтаксиса
- **Markdown** — через безопасный markdown renderer
- **Image** — data URL preview
- **PDF** — iframe preview
- **Binary unknown** — сообщение о недоступности preview + download

#### Поддерживаемые форматы в workspace backend

**Текстовые / кодовые:**
`txt`, `md`, `markdown`, `json`, `js`, `jsx`, `ts`, `tsx`, `css`, `scss`, `html`, `htm`, `xml`, `yml`, `yaml`, `csv`, `py`, `java`, `c`, `cpp`, `h`, `go`, `rs`, `rb`, `php`, `sh`, `sql`, `env`, `ini`, `toml`, `log`, `vue`, `svelte`, `config`, `gitignore`, `editorconfig`, `dockerfile`, `ps1`, `bat`.

**Изображения:**
`png`, `jpg`, `jpeg`, `gif`, `webp`, `bmp`, `svg`, `ico`.

**Документы preview:**
`pdf`.

**Архивы:**
`zip`, `tar`, `tgz`, `tar.gz`.

#### Ограничения и квоты
- общий размер workspace задаётся через `WORKSPACE_QUOTA_MB`;
- максимальный одиночный файл — через `WORKSPACE_MAX_FILE_MB`;
- preview текста ограничен `200 KB`;
- grep-search читает только текстовые файлы до `512 KB`;
- история хранит до `30` ревизий на файл.

#### Где смотреть реализацию
- UI: `src/components/Workspace.jsx`, `src/components/FileTree.jsx`, `src/components/FilePreview.jsx`
- client API: `src/lib/workspace.js`
- backend: `server/workspace.js`, `server/index.js`

### 13. AI-функции внутри Workspace

#### Назначение
Этот подмодуль превращает Workspace из простого файлового дерева в рабочий инструмент для правки проекта через модель.

#### Реализованные сценарии

| Функция | Что делает |
|---|---|
| AI Create File | Просит путь + инструкцию и генерирует содержимое нового файла |
| AI Apply Patch | Просит инструкцию, получает patch JSON, применяет `search/replace` к текущему текстовому файлу |
| Attach to chat | Берёт файл из workspace и прикладывает его как attachment в чат |

#### Ограничения
- AI Patch доступен только для текстовых файлов;
- patch должен найти точный `search` fragment в текущем тексте;
- модель не делает shell/run/build изнутри приложения.

### 17. Android wrapper

#### Назначение
Даёт Android-пользователю установить BrowserAI как приложение, не переписывая web UI нативно.

#### Что умеет Android-приложение

| Функция | Описание |
|---|---|
| WebView app shell | Открывает Railway URL как приложение |
| Cookies | Поддерживает пользовательскую web-сессию |
| File chooser | Позволяет выбирать файлы из Android |
| Download manager | Скачивает файлы в системную папку Downloads |
| Update check | Проверяет latest GitHub Release |
| OTA APK download | Скачивает APK и открывает системный installer |
| Offline error screen | Показывает сообщение при отсутствии сети / недоступности backend URL |

#### Что важно помнить разработчику
- Android-приложение не заменяет backend;
- SQLite, auth, workspace, cloud sync остаются на Railway;
- все web-исправления приезжают в Android «по воздуху» после redeploy;
- APK нужно пересобирать только при изменении `android-app/`.

#### Где смотреть реализацию
- `android-app/app/src/main/java/ai/browser/app/MainActivity.java`
- `android-app/app/src/main/AndroidManifest.xml`
- `android-app/app/build.gradle`

### 20. Основные сценарии использования

1. Пользователь входит в аккаунт.
2. Добавляет API-ключ и выбирает модель.
3. Ведёт диалог в чате.
4. При необходимости включает Web AI.
5. Прикладывает файлы к сообщению или выбирает файл из workspace.
6. Работает с workspace как с файловой областью проекта.
7. Создаёт новые файлы или патчит существующие через AI.
8. Синхронизирует историю и настройки между desktop и Android.
9. При необходимости шифрует ключи мастер-паролем.
